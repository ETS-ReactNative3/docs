## What for initialize

Chatty sdk를 정상적으로 사용하기 위해서는 아래 두가지가 필요합니다

- dashboard에서 생상한 app의 API Key
- 로그인된 사용자의 기본정보 (id, name) 을 채팅을 위한 Member로 사용하겠다고 지정

## Where to initialize
- 앱이 bootup 하는 과정에 로그인된 사용자를 식별가능한 시점에 init 함수를 호출합니다.

### 특징
- 중복으로 호출이 되어도 member를 중복으로 생성하지않습니다. member는 id 값으로 유일하게 존재합니다
- 멤버가 당신의 서비스에서 탈퇴후 재가입시 id값은 달라져있어야 합니다.(social login 과 같은 경우를 사용할경우 provider's user id값을 사용하지 마세요)


file: App.js
```javascript
import Chatty, { ChattyTypes } from 'chatty-cloud';

...

if (isLoggedIn) {
  await Chatty.init({
    apiKey: <API KEY>, // API Key of your app that created in dashboard 
    member: {
      id: <MEMBER ID>,                // mandatory  (ex: user.id)
      name: <MEMBER NAME>,            // mandatory  (ex: user.name)
      avatar: <MEMBER AVATAR URL>,    // optional   (ex: user.avatar)
      language: <MEMBER LANGUAGE>,    // optional   (ex: 'en')
      country: <MEMBER COUNTRY>,      // optional   (ex: 'US')
      deviceToken: <FCM DEVICE TOKEN> // optional   
    }
  });
}
```

<InfoBlock type="warning">
  Please manage API Key securely.
</InfoBlock>




## Email address and password

This is the only authentication mechanism where ChattyCloud maintains user credentials.

Additionally, users can opt to configure [two-factor authentication (2FA)](/concepts/mfa). This option requires **something you know** _(i.e. your password)_ and **something you have** _(i.e. recovery codes)_.

## Single sign-on

Users can authenticate with their chosen corporate identity provider _(i.e. Okta)_ instead of maintaining passwords with ChattyCloud.

Once [SSO](/concepts/sso) is enabled for an `organization`, all members are redirected through that identity provider's authentication flow. Moving forward, they must pass through SSO to access their ChattyCloud account.

## OAuth via GitHub

Users can authenticate with ChattyCloud using their GitHub account.

<InfoBlock type="warning">
  ChattyCloud doesn't maintain the passwords for these accounts. Losing access
  to your GitHub account prevents accessing your ChattyCloud account.
</InfoBlock>


<InfoBlock type="note">
  Enabling SSO removes OAuth access for all members of your{' '}
  <em>organization</em>, meaning they will no longer be able to sign in with
  their GitHub credentials.
</InfoBlock>
